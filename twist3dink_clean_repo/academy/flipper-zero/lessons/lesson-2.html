    Understanding Rolling Codes

    Rolling codes (also called hopping codes) are a security mechanism used in modern remote entry systems to prevent replay attacks. Understanding how they work is crucial for security research.

    How Rolling Codes Work

    Instead of sending the same static code every time, rolling code systems generate a new code for each transmission:

    1. Synchronized Counter: Both the remote and receiver maintain a counter
    2. Algorithm: Each uses the same algorithm to generate the next valid code
    3. Transmission: Remote sends current code and increments counter
    4. Validation: Receiver checks if code matches expected value
    5. Rejection: Old codes are rejected, preventing replay attacks

    üí° Common Rolling Code Systems

    • KeeLoq: Used in many car key fobs and garage doors
    • HCS (Hopping Code System): Microchip's implementation
    • Proprietary Systems: Many manufacturers use custom algorithms

    Can You Defeat Rolling Codes?

    Simple replay attacks don't work against rolling codes, but there are some known weaknesses:

    • RollJam Attack: Capture two codes by jamming the first, replay later (requires special hardware)
    • Code Prediction: Some older implementations have weak algorithms
    • Brute Force: Theoretically possible but impractical (billions of combinations)
    • Signal Cloning: Not possible with genuine rolling code systems

    ‚ö†Ô∏è Rolling Code Ethics

    Attempting to defeat rolling code systems on vehicles or buildings you don't own is illegal. These security mechanisms exist for good reason. Focus your research on understanding the technology, not bypassing it maliciously.

    Identifying Rolling Code Devices

    When you capture a signal, here's how to tell if it uses rolling codes:

    • Capture the same button press multiple times
    • Compare the captured data
    • Static Code: Data is identical each time
    • Rolling Code: Data changes with each press
    • Replay won't work if the receiver has already processed newer codes

    Common Sub-GHz Vulnerabilities

    Understanding common weaknesses helps you know what to test for and how to protect against them.

    1. Static Code Transmission

    Vulnerability: Device sends the same code every time

    Impact: Simple replay attacks work indefinitely

    Mitigation: Upgrade to rolling code systems

    2. Weak Rolling Code Implementation

    Vulnerability: Predictable algorithms or short code length

    Impact: Codes can be predicted or brute forced

    Mitigation: Use industry-standard implementations (KeeLoq, AES)

    3. Excessive Range

    Vulnerability: Signals can be captured from long distances

    Impact: Attackers can capture codes from parking lots, streets

    Mitigation: Reduce transmission power, require strong signal at receiver

    4. No Encryption

    Vulnerability: Data transmitted in clear text

    Impact: Anyone can capture and understand the signal

    Mitigation: Implement encryption (AES-128 or better)

    5. Lack of Authentication

    Vulnerability: Receiver accepts any valid-looking signal

    Impact: Forged signals can trigger the device

    Mitigation: Implement proper authentication and pairing

    6. Jamming Susceptibility

    Vulnerability: Continuous interference prevents legitimate use

    Impact: Denial of service attacks possible

    Mitigation: Frequency hopping, backup access methods

    Understanding Sub-GHz Radio

    Sub-GHz radio refers to wireless communications that operate below 1 GHz (1000 MHz). These frequencies are used extensively in everyday devices because they offer excellent range and can penetrate walls and obstacles better than higher frequencies like WiFi or Bluetooth.

    The Flipper Zero can receive, analyze, and transmit on several Sub-GHz frequency bands, making it an excellent tool for security testing wireless access control systems, remote controls, and IoT devices.

    ‚ö†Ô∏è Critical Legal Warning

    Radio frequency regulations vary by country and are strictly enforced.

    • In the US, FCC regulations prohibit unauthorized transmission on many frequencies
    • In Europe, ETSI standards regulate radio usage
    • Jamming or interfering with licensed frequencies is illegal everywhere
    • Only capture and replay signals from devices you own or have explicit permission to test
    • Never transmit on frequencies you're not authorized to use

    Common Sub-GHz Applications

    • Garage Door Openers: 300-390 MHz (most common: 315 MHz and 390 MHz)
    • Car Key Fobs: 315 MHz (North America), 433.92 MHz (Europe/Asia)
    • Wireless Doorbells: 315 MHz, 433.92 MHz
    • Weather Stations: 433.92 MHz, 868 MHz, 915 MHz
    • Tire Pressure Monitors: 315 MHz, 433.92 MHz
    • Home Security Sensors: 315 MHz, 433.92 MHz, 868 MHz
    • Remote Controls: Various frequencies depending on device and region

    Supported Frequency Bands

    The Flipper Zero supports different frequency ranges depending on your region. Here's what's available:

    Frequency Common Uses Region
    300-348 MHz Garage doors, car alarms North America
    387-464 MHz Car keys, remotes, sensors Worldwide
    779-928 MHz ISM band devices, LoRa, weather stations Region-specific

    üí° ISM Bands Explained

    ISM (Industrial, Scientific, Medical) bands are portions of the radio spectrum reserved for unlicensed use. Common ISM bands include:

    • 433 MHz ISM: Europe and most of the world
    • 868 MHz ISM: Europe
    • 915 MHz ISM: North America

    These bands are popular for IoT devices, smart home products, and wireless sensors because they don't require licensing.

    Understanding Modulation

    Modulation is how information is encoded onto a radio wave. Think of it as different ways to "write" data onto a carrier frequency. Understanding modulation is crucial for successfully capturing and replaying signals.

    Common Modulation Types

    OOK (On-Off Keying)

    How it works: The simplest form of modulation. The carrier is either ON (transmitting) or OFF (not transmitting).

    Used in:

    • Simple garage door openers
    • Basic wireless doorbells
    • Older car key fobs
    • Simple remote controls

    Easiest to capture and replay because of its simplicity.

    ASK (Amplitude Shift Keying)

    How it works: Similar to OOK but uses different amplitude levels instead of just on/off. Can encode more data.

    Used in:

    • RFID systems
    • Some wireless sensors
    • Modern remote controls

    Moderate difficulty to capture and replay.

    FSK (Frequency Shift Keying)

    How it works: Data is encoded by shifting between two or more frequencies. More robust against noise.

    Used in:

    • Modern car key fobs
    • Wireless security systems
    • IoT sensors
    • Weather stations

    More complex but Flipper handles it automatically.

    üí° Flipper's Auto-Detection

    The Flipper Zero automatically detects and decodes most common modulation types. When you capture a signal, it will identify whether it's OOK, FSK, or another type and configure itself accordingly. You don't need to manually specify modulation in most cases.
