    ⌨️ Introduction to BadUSB

    What is BadUSB?

    BadUSB is an attack technique where a USB device impersonates a keyboard or mouse to send malicious commands to a computer. Since computers trust keyboards by default, this attack bypasses many security measures.

    The Power of Keyboard Emulation

    Why BadUSB is Dangerous:

✓ No malware needed - Just keystrokes
✓ Bypasses antivirus - Looks like legitimate typing
✓ Fast execution - Can run in seconds
✓ Cross-platform - Works on Windows, Mac, Linux
✓ Pre-boot attacks - Can even modify BIOS
✓ Physical access = game over

    Historical Context

    The BadUSB Vulnerability:

    Discovered in 2014 by security researchers Karsten Nohl and Jakob Lell. They demonstrated that USB device firmware can be reprogrammed to:

    • Emulate keyboards and inject keystrokes
    • Redirect network traffic
    • Install backdoors
    • Modify files during transfer

    The scary part: This vulnerability is built into the USB specification itself and cannot be patched!

    How BadUSB Attacks Work

    Attack Flow:
┌─────────────────────────────────────┐
│ 1. Device plugged into USB port     │
│ 2. Computer recognizes it as keyboard│
│ 3. Device sends pre-programmed keys │
│ 4. Commands execute automatically   │
│ 5. Payload delivered in seconds     │
│ 6. Device can eject itself (stealth)│
└─────────────────────────────────────┘

Example Attack Timeline:
0s  - Flipper plugged in
1s  - Recognized as USB keyboard
2s  - Opens command prompt/terminal
3s  - Types commands
5s  - Executes payload
7s  - Cleans up traces
8s  - Attack complete

    Common BadUSB Payloads

    What Can BadUSB Do?
    • Reverse Shells: Remote access to the target system

    Timing with DELAY

    DELAY is crucial for reliable scripts. Computers need time to open windows, load programs, and process commands.

    Timing Best Practices:

DELAY 1000      → 1 second (1000 milliseconds)
DELAY 500       → Half second
DELAY 2000      → 2 seconds

When to use DELAY:
• After opening programs (500-2000ms)
• After GUI commands (300-500ms)
• Before typing URLs (500-1000ms)
• After pressing ENTER (100-300ms)
• Slow computers need longer delays!

Too fast = Script fails
Too slow = Takes forever but reliable

    🛡️ Defending Against BadUSB

    Why BadUSB is Hard to Stop

    The Fundamental Problem:

    BadUSB exploits trust in Human Interface Devices (HID). Computers must trust keyboards - otherwise, how would you use your computer? This makes BadUSB attacks extremely difficult to prevent completely.

    Physical Security

    First Line of Defense:
    • Lock your screen: Always when stepping away (Windows+L, Ctrl+Alt+L)
    • Disable USB ports: Glue, port blockers, BIOS settings
    • Monitor USB usage: Be aware of what's plugged in
    • Physical access control: Secure work areas, badge access
    • Cable locks: Prevent device removal and USB access
    • Clear desk policy: No unattended computers

    Technical Defenses

    Software Protections:

USB Device Control:
• Windows: Device Installation restrictions
• macOS: System Preferences → Security
• Linux: USBGuard, udev rules
• Enterprise: Manage via Group Policy

Endpoint Protection:
• EDR (Endpoint Detection & Response)
• Behavioral analysis (detect rapid typing)
• Process monitoring (catch malicious commands)
• Network monitoring (detect exfiltration)

Application Whitelisting:
• Only allow approved programs to run
• Blocks payload-launched executables
• Examples: AppLocker, SELinux

Disable Auto-Run:
• Prevent automatic execution from USB
• Group Policy: Turn off Autoplay
• Registry modifications
