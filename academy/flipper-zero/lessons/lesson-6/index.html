What is BadUSB?

BadUSB is a security exploit that takes advantage of the trust computers place in USB devices. When you plug in a USB keyboard, your computer automatically trusts it and accepts whatever keystrokes it sends. BadUSB attacks exploit this by making a device pretend to be a keyboard and then rapidly typing commands to compromise the system.

USB Human Interface Device (HID) Attacks

Human Interface Devices (HIDs) are peripherals like keyboards, mice, and game controllers. Operating systems trust HIDs implicitly because they're essential for user interaction. This trust creates a security vulnerability:

  • When you plug in a keyboard, the OS doesn't ask "should I trust this keyboard?"
  • It immediately starts accepting keystrokes
  • The keyboard could type anything - including malicious commands
  • The OS has no way to distinguish between human keystrokes and automated ones

Flipper Acts as Keyboard/Mouse

When you connect Flipper Zero to a computer via USB and run a BadUSB script, Flipper identifies itself as a keyboard (and optionally mouse). The computer sees it as a legitimate input device and processes whatever "keystrokes" Flipper sends. This happens in milliseconds, often faster than a human could react.

Automated Keystroke Injection

BadUSB scripts can:

  • Open applications (calculator, terminal, web browser)
  • Execute system commands
  • Download and run files from the internet
  • Modify system settings
  • Extract data and upload it to remote servers
  • Disable security features

All of this happens automatically when the script runs, without any user interaction beyond plugging in the USB device.

Legitimate Uses vs Malicious Uses

Legitimate uses of BadUSB:

  • Penetration testing: Authorized security assessments of computer systems
  • IT automation: Rapid deployment of configurations across multiple machines
  • Personal productivity: Automating repetitive tasks on your own computers
  • Education: Learning about USB security vulnerabilities
  • Accessibility: Creating custom input solutions for users with disabilities

Malicious uses (illegal and unethical):

  • Unauthorized access to computer systems
  • Data theft or corporate espionage
  • Installing malware or ransomware
  • Disabling security systems without permission
  • Any use on computers you don't own or have authorization to test

Creating BadUSB Scripts

BadUSB scripts use Ducky Script, a simple scripting language originally developed for the USB Rubber Ducky penetration testing tool.

Ducky Script Language Basics

Ducky Script is designed to be human-readable and straightforward. Scripts are plain text files with a .txt extension, stored in the /badusb/ folder on your Flipper's SD card.

Basic structure:

  • Each line represents one command or keystroke
  • Commands are case-sensitive
  • Comments start with REM (remark)
  • Scripts execute line-by-line from top to bottom

Common Commands

DELAY - Pause execution for specified milliseconds

DELAY 1000 REM Waits for 1 second (1000 milliseconds)

STRING - Type out text exactly as written

STRING Hello World REM Types the text "Hello World"

ENTER - Press the Enter/Return key

ENTER REM Presses Enter key once

GUI - Windows key (Windows), Command key (Mac), Super key (Linux)

GUI r REM Windows: Opens Run dialog (Win+R) REM Mac: Opens Spotlight (Cmd+R)

ALT - Alt key modifier

ALT F4 REM Closes the active window

CTRL - Control key modifier

CTRL c REM Copies selected text

Other useful keys: SHIFT, TAB, ESCAPE, BACKSPACE, DELETE, UP, DOWN, LEFT, RIGHT, HOME, END, PAGEUP, PAGEDOWN

Script Structure

A well-structured BadUSB script includes:

  1. Header comments: Description of what the script does
  2. Initial delay: Give the system time to recognize the USB device
  3. Command sequence: Your actual automation steps
  4. Cleanup: Close windows or return system to normal state
REM Title: Example Script Template REM Description: What this script does REM Author: Your name REM Target: Windows 10/11 REM Wait for device recognition DELAY 2000 REM Your commands here GUI r DELAY 500 STRING notepad ENTER DELAY 1000 STRING Script executed successfully! REM End of script

Understanding GPIO

GPIO (General Purpose Input/Output) pins are one of Flipper Zero's most versatile features, opening up possibilities for hardware interaction and custom projects.

General Purpose Input/Output Pins

GPIO pins are digital interfaces that can be programmed to:

  • Read signals (Input): Detect button presses, sensor readings, or signals from other devices
  • Send signals (Output): Control LEDs, relays, motors, or other electronic components
  • Communicate (Protocols): Exchange data using UART, SPI, or I2C protocols

Reading Sensors

GPIO input mode allows Flipper to read data from various sensors:

  • Temperature sensors: DHT11, DHT22, DS18B20
  • Motion sensors: PIR (passive infrared) detectors
  • Light sensors: Photoresistors, photodiodes
  • Buttons and switches: Simple on/off inputs
  • Distance sensors: Ultrasonic or infrared ranging

Controlling LEDs and Devices

GPIO output mode lets Flipper control external hardware:

  • LEDs: Turn on/off, create patterns, brightness control (with PWM)
  • Relays: Switch high-power devices on/off
  • Servos: Control mechanical movement
  • Displays: Drive small OLED or LCD screens
  • Buzzers: Create tones and alerts

Pin Layout and Functions

Flipper Zero has 18 GPIO pins arranged in two rows. Key pins include:

  • Power pins: 3.3V output (for powering sensors/components) and Ground (GND)
  • Digital pins: Can be configured as input or output
  • UART pins: RX (receive) and TX (transmit) for serial communication
  • SPI pins: CLK (clock), MOSI (master out), MISO (master in), CS (chip select)
  • I2C pins: SDA (data) and SCL (clock)
Pin Reference: Always check the official Flipper Zero documentation for the exact pin layout before connecting hardware. Incorrect connections can damage Flipper or your components.

GPIO Basics

Working with GPIO requires understanding electrical fundamentals and proper connection techniques.

Pin Modes: Input, Output, UART, SPI, I2C

Input Mode:

  • Pin reads external voltage levels
  • Detects HIGH (3.3V) or LOW (0V) states
  • Used for buttons, switches, and sensors
  • Often requires pull-up or pull-down resistors

Output Mode:

  • Pin generates voltage to control external devices
  • Can be set HIGH (3.3V) or LOW (0V)
  • Used to drive LEDs, relays, and other components
  • Limited current output (typically 20mA per pin)

UART (Universal Asynchronous Receiver/Transmitter):

  • Serial communication protocol
  • Two wires: TX (transmit) and RX (receive)
  • Used for debugging, GPS modules, serial devices
  • Configurable baud rate (speed)

SPI (Serial Peripheral Interface):

  • Fast, synchronous communication
  • Four wires: CLK, MOSI, MISO, CS
  • Used for displays, SD cards, sensors
  • Supports multiple devices on same bus

I2C (Inter-Integrated Circuit):

  • Two-wire communication protocol
  • SDA (data) and SCL (clock)
  • Multiple devices can share the same bus
  • Common in sensors, displays, and expansion modules

Voltage Levels (3.3V)

Flipper Zero operates at 3.3V logic levels. This is critical to remember:

  • 3.3V is HIGH: Logic "1" or "on"
  • 0V (GND) is LOW: Logic "0" or "off"
  • Do NOT connect 5V devices directly - they can damage Flipper's GPIO pins
  • Use level shifters when interfacing with 5V Arduino or other 5V devices

Ground and Power Pins

Proper power and ground connections are essential:

  • Ground (GND): Common reference point for all signals, must be connected between Flipper and external devices
  • 3.3V Power: Can provide limited power to external circuits (check maximum current draw)
  • 5V Power: Available from USB when connected, can power higher-current devices

Protection Circuits

While Flipper has some built-in protection, it's not invincible:

  • Pins have limited current capacity - use resistors with LEDs
  • No reverse polarity protection - connecting power backwards can cause damage
  • ESD (electrostatic discharge) can damage sensitive circuits
  • Always double-check connections before applying power

The classic "Hello World" of electronics: blinking an LED.

UART Communication

UART allows Flipper to communicate with other devices serially.

Common UART applications:

  • Debug consoles: Access device debug output

  • Common GND between both devices
  • Match baud rates (9600, 115200, etc.)

Safety with GPIO

Working with hardware requires caution to avoid damaging your Flipper or connected components.

Never Connect to Higher Voltages

Flipper's GPIO pins are designed for 3.3V logic. Connecting higher voltages will cause permanent damage:

  • 5V devices: Use a level shifter or voltage divider
  • 12V systems: Use optocouplers or relays for isolation
  • AC voltage: Never connect directly - use appropriate isolation and conversion

There is no "maybe it will work" with overvoltage - it WILL damage the pins immediately and irreversibly.

Check Polarity

Many components are polarity-sensitive:

  • LEDs: Anode (long leg) to positive, cathode (short leg) to ground
  • Electrolytic capacitors: Negative terminal clearly marked, wrong polarity can cause explosion
  • Diodes: Current flows one direction only
  • Sensors: VCC/Power and GND connections must be correct

Use Current-Limiting Resistors

Resistors protect both Flipper and your components:

  • LEDs: 220Œ©-330Œ© resistor prevents overcurrent
  • Pull-up/Pull-down: 10kŒ© typical for button inputs
  • Base resistors: For transistor circuits

Calculate proper resistor values using Ohm's Law: R = V / I

Avoid Shorts

Short circuits can destroy GPIO pins or damage Flipper's power supply:

  • Never connect power directly to ground
  • Check for stray wires or metal objects on your breadboard
  • Use jumper wires carefully - loose connections can cause shorts
  • Power off before making connection changes
