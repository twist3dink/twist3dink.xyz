Lesson 2: Setup + File Ops (So You Don’t Lose Your Work)

Estimated time: 0 minutes | XP Reward: 50

Objective

By the end of this lesson you will have a stable setup, a basic backup, and you’ll successfully transfer a file to/from your Flipper Zero. This is the foundation that keeps every later lesson from turning into chaos.

Rules (non-negotiable):
  • Only test on devices you own or have explicit permission to test.
  • We are using official/standard workflows for this track. Custom firmware is out of scope here.

Prereqs (2 minutes)

  • Flipper Zero + USB-C cable
  • MicroSD card installed (recommended; most users should have one)
  • A computer with USB (Windows/macOS/Linux)
  • Battery above 30%
Important: Do not interrupt firmware updates. If you start an update, finish it.

What Exists Inside the UI (Fast Map)

You don’t need to memorize everything. You need to know where to check device info, where files live, and how to get back home without panicking.

Two spots you must be able to find

  • Device Info: Settings → Device Info (firmware version, storage, etc.)
  • File Browser: Browse files on internal storage / SD card
Navigation tip: If you get lost, press Back until you’re somewhere familiar. The UI is forgiving.

Step-by-Step: Setup + Backup + Transfer

Step 1 — Verify device info (3–5 minutes)

  1. On Flipper: go to Settings → Device Info.
  2. Write down (or take a photo of) your Firmware version and Storage status.

Step 2 — Confirm your SD card is usable (3–8 minutes)

  1. Open the File Browser.
  2. Confirm you can see the SD card and browse folders.
  3. If the SD card is missing or errors out, jump to the Troubleshooting section.

Step 3 — Create a backup folder on your computer (2 minutes)

Create a folder on your computer (anywhere you want), named exactly:

  • flipper-backup

Step 4 — Connect and transfer ONE test file (10 minutes)

  1. Connect the Flipper to your computer via USB.
  2. Using your standard file transfer method (qFlipper or file access), create a plain text file on your computer named:
    • hello-flipper.txt
  3. Put one line of text inside it, e.g. “Lesson 2 transfer test”.
  4. Transfer hello-flipper.txt onto the Flipper (any folder you can easily find again).
  5. Then transfer the same file back from Flipper → computer into your flipper-backup folder.

Step 5 — Make a minimal backup (5–10 minutes)

Copy the following from Flipper storage into flipper-backup (whatever exists on your device):

  • /infrared/ (or IR files)
  • /nfc/
  • /rfid/
  • /subghz/
  • /badusb/
Why this matters: When you start collecting captures, you will eventually want to organize, rename, and experiment. Backups keep you bold.

Deliverables (Proof Pack)

To finish Lesson 2, you must produce these artifacts. No artifacts = not finished.

  • D2.1 — Photo/screenshot of Settings → Device Info showing firmware version.
  • D2.2 — A computer folder named flipper-backup containing:
    • hello-flipper.txt (the file that made a round trip)
    • Copies of any of the module folders/files listed in Step 5 that exist on your Flipper
  • D2.3 — A short note (2–5 sentences) answering:
    • Where did you place hello-flipper.txt on the Flipper?
    • What was the hardest part of transfer/backup for you?

Verification

Check these boxes. If you can’t honestly check a box, you’re not done yet.

  • I can find Device Info on the Flipper without guessing.
  • I can browse the File Browser and locate the folder where I put hello-flipper.txt.
  • hello-flipper.txt successfully transferred computer → Flipper → computer.
  • My flipper-backup folder exists and contains my files.

Troubleshooting (common failures)

Flipper not detected / transfer tool can’t connect

  • Try a different USB cable (many “charge-only” cables exist).
  • Try a different USB port.
  • Reboot Flipper (power menu) and reconnect.

SD card not showing / errors

  • Reseat the SD card.
  • Confirm it’s formatted correctly (FAT32 is commonly required).
  • If you must format, back up anything important first.

You can’t find the file you transferred

  • Search the folder you chose slowly; don’t assume it “didn’t transfer.”
  • Use a single obvious location for now (we’re training consistency).
Do not “fix it” by installing random firmware or mystery tools. If the official workflow is failing, we diagnose the workflow. We don’t escalate into side quests.

Finished Means

  • I produced D2.1, D2.2, and D2.3.
  • I can repeat the transfer workflow without re-reading this page.
  • I understand where my future captures will be saved and how to back them up.
