Lesson 5: Sub-GHz — Receive + Label + Interpret

Estimated time: 25–40 minutes | XP Reward: 50

Objective

Capture a sub-GHz signal from a device you own, save it, label it correctly, and write a short analysis of what you captured and whether replay would plausibly work. The win condition here is understanding + documentation, not “making things open.”

Boundary (non-negotiable):
  • Only capture signals from devices you own or have explicit permission to test.
  • No experimenting on public/neighbor devices. Not even “just to see.”
  • No jamming, no interference. If you don’t know what that means, good — don’t learn it here.

What Sub-GHz is (one-minute version)

Sub-GHz is radio communication below 1GHz. Common consumer devices use it for short-range control and sensors. The same band can be used by many different protocols, some replayable (fixed code) and some not (rolling code / challenge-response).

Common examples (pick one you own)

  • Wireless doorbell (often 433MHz)
  • Simple RF outlet remote
  • Weather station sensor
  • Garage remotes / car fobs exist, but many are rolling code (hard-mode, not required)
Best target for this lesson: a cheap wireless doorbell or RF outlet remote. You want a clean first capture, not a crypto puzzle.

Prereqs

  • Flipper Zero
  • One sub-GHz transmitting device you own (remote/sensor)
  • Your region set correctly on the Flipper (Settings → Region) so it behaves within legal limits
Safety setup: Do this indoors, away from other similar devices if possible. You’re reducing accidental cross-effects.

Step-by-step: Find → Capture → Save

Step 1 — Find the frequency (Frequency Analyzer)

  1. Open Sub-GHz → Frequency Analyzer.
  2. Press a button on your remote (or trigger your sensor if possible).
  3. Note the strongest frequency shown (example: 433.92 MHz).
If you can’t get a reading: move closer, press again, and keep the remote within 6–12 inches of the Flipper.

Step 2 — Capture the signal (Read or Read RAW)

Use the simplest approach first:

  • Try Sub-GHz → Read (decoded) first.
  • If it doesn’t recognize it, use Read RAW.
  1. Set the frequency to what you found (or let it auto-detect if available).
  2. Start listening.
  3. Press the remote button once.
  4. When the Flipper detects it, save it with a meaningful name.

Step 3 — Save with a strict naming format

Name it like this (copy the pattern):

  • device-function-frequency
  • Example: doorbell-front-433_92
  • Example: rfoutlet-on-315_00
Why this matters: your library will become a graveyard if you name everything “test1”.

Step 4 — Confirm it saved and open the file info

  1. Go to Sub-GHz → Saved.
  2. Find your saved entry and open it.
  3. Confirm you can see details like frequency and modulation/protocol (whatever the Flipper shows).

Deliverables (Proof Pack)

To finish Lesson 5, you must produce these artifacts.

  • D5.1 — One saved Sub-GHz capture with the strict naming format.
  • D5.2 — A screenshot/photo of:
    • Frequency Analyzer result or
    • The saved capture details screen (showing frequency/protocol)
  • D5.3 — A short “Capture Report” (5–10 lines) answering:
Capture Report template:
  • Device: (what you captured)
  • Ownership: Mine / I have permission
  • Frequency:
  • Mode used: Read or Read RAW
  • Protocol/modulation shown: (or “unknown”)
  • What the button does: (doorbell, toggle outlet, etc.)
  • Rolling code suspected? Yes/No/Unknown (explain why)
  • Would replay likely work? Likely / Unlikely / Unknown (explain in one line)
Note: You are not required to replay anything for this lesson. We are learning capture + analysis without creating bad habits.

Verification

  • I captured a sub-GHz signal from a device I own.
  • I saved it with the strict naming format.
  • I can find it again under Sub-GHz → Saved.
  • I completed the Capture Report honestly.

Troubleshooting

Frequency Analyzer shows nothing

  • Move the remote closer (6–12 inches).
  • Press the button once per second (don’t mash).
  • Try a different device you own (doorbell/RF outlet remotes are often easiest).

Read mode doesn’t detect anything

  • Switch to Read RAW.
  • Double-check region settings are correct.
  • Confirm you’re on the right band (315/433/868/915 depend on the device and region).

It detects but won’t save / file missing

  • Check SD card is present and working (Lesson 2 setup).
  • Try saving with a shorter name (some systems are picky).
Good judgment: If you’re working in a dense apartment building and you see “lots of stuff,” stop and pick a safer environment. You’re responsible for not affecting others.

Finished Means

  • D5.1 + D5.2 + D5.3 exist.
  • You can repeat: Frequency Analyzer → Read/RAW → Save → Find saved → Write report.
