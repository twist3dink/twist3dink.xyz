Lesson 4: NFC/RFID — Read + Document Your Tags

Estimated time: 0 minutes | XP Reward: 50

Objective

Read at least two tags/cards you personally own (NFC and/or 125kHz RFID), save the reads, and document what you learned in a simple “Tag Report.” This lesson is about understanding what the Flipper can see — not trying to get into places.

Boundary (non-negotiable):
  • Only scan tags/cards you own or have explicit permission to test.
  • We are not practicing “emulation to access systems” in this track. If you’re authorized to use your own badge, that’s your real life — but it’s not a required exercise here.

Quick mental model (what matters)

Two buckets

  • 125kHz RFID (low frequency): Usually simpler. Often an ID/number broadcasted repeatedly.
  • 13.56MHz NFC (high frequency): More complex. Can have memory sectors and authentication.

What you’re trying to learn

  • Can the Flipper identify the type (NFC vs RFID, and the family)?
  • What public identifiers are visible (UID, ATQA/SAK, etc.)?
  • Does it show protected sectors or limited access?
Reality check: “Not readable beyond UID” is not a failure. That’s modern security doing its job.

Prereqs

  • Flipper Zero
  • At least 2 items you own, for example:
    • NFC sticker/tag
    • Gym card you own
    • Transit card you own
    • Key fob that is yours (125kHz or NFC)
Best starter combo: 1 NFC sticker + 1 random RFID/NFC fob you own. Transit/payment cards can be finicky; don’t pick hard-mode first.

Step-by-step: Read + Save (two items)

Step 1 — Read item A (try NFC first)

  1. Go to NFC → Read.
  2. Hold item A to the back of the Flipper. Hold steady for 1–2 seconds.
  3. If it reads: write down what it says (type/UID/etc.) and Save with a meaningful name.
  4. If NFC shows nothing: try 125kHz RFID → Read for the same item A.

Step 2 — Read item B

  1. Repeat the same process for item B.
  2. Save item B with a meaningful name.

Step 3 — Confirm saved files (proof you can find them)

  1. Go to NFC → Saved and/or RFID → Saved.
  2. Confirm both saved entries are visible.
  3. Open each saved entry and verify it shows the same info you saw when reading.

Deliverables (Proof Pack)

To finish Lesson 4, create these artifacts. Keep it simple and auditable.

  • D4.1 — Two saved reads on the Flipper (Item A + Item B) with meaningful names.
  • D4.2 — Two “Tag Reports” (copy/paste template below into your notes app or a file):
Tag Report template (copy this twice):
  • Tag name: (your saved file name)
  • Ownership: Mine / I have permission
  • Type: NFC 13.56MHz or RFID 125kHz
  • What Flipper identified: (card/tag family)
  • Visible identifiers: (UID, ATQA/SAK, etc. — whatever is shown)
  • Protected/locked? Yes/No/Unknown (based on what Flipper reports)
  • Notes: 1–3 lines (anything weird, any errors, placement tips)
  • D4.3 — One photo/screenshot per item showing the read result or the saved entry screen.

Verification

  • I can read and save two tags/cards I own.
  • I can find them again under Saved.
  • I created two Tag Reports + collected proof screenshots/photos.
  • I can explain in one sentence the difference between 125kHz RFID and 13.56MHz NFC.

Troubleshooting

It won’t read at all

  • Move the card slowly around the back of the Flipper — antenna position matters.
  • Hold steady for 1–2 seconds (don’t tap-and-run).
  • Try the other module (NFC vs 125kHz RFID).

It reads but only shows minimal info

  • That’s normal for many secure implementations.
  • Your deliverable is “document what’s visible,” not “break encryption.”

It says protected / keys required / partial read

  • Also normal. Record that outcome in the Tag Report.
  • Do not chase key extraction here — out of scope for this track.
Do not test random badges you find in public. If you don’t own it, it’s not “practice,” it’s evidence.

Finished Means

  • D4.1 + D4.2 + D4.3 exist.
  • You can repeat read → save → find saved without help.
